# 🏃 运动小英雄（Rungogo App）

一个面向儿童的运动打卡小游戏，采用卡通化的星球地图形式，把每天运动变成“点亮星球”的冒险。

**在线体验**：https://patrickz2.github.io/rungogo/

---

## ✨ 功能介绍

### 1. 创建英雄形象
- 上传头像照片，自动生成圆形卡通头像
- 输入专属昵称，成为“运动小英雄”
- 首次设置后自动保存，下次打开无需重复设置

### 2. 运动打卡
- 支持 9 种运动类型：足球、篮球、跳绳、跑步、游泳、俯卧撑、仰卧起坐、骑车、自定义
- 上传运动照片，记录运动数据（如跑步距离/用时、跳绳次数等）
- 可填写运动感想，生成运动日记
- 打卡成功触发粒子庆祝动画和勋章解锁

### 3. 运动星球地图
- 6 颗主题星球：足球星、篮球星、跳跃星、跑步星、游泳星、力量星
- 根据打卡次数解锁星球，进度条可视化点亮进度
- 点击星球可查看该项目的所有打卡历史

### 4. 运动日记
- 按时间倒序展示每次打卡记录
- 显示运动照片、运动类型、运动数据、当天感想

### 5. 勋章系统
- 连续打卡 3/7/30 天勋章
- 累计打卡 20/50 次勋章
- 完成 5 种不同运动勋章
- 单项运动累计勋章

### 6. 个人中心
- 总打卡次数、连续天数、运动种类统计
- 最近 7 天打卡柱状图
- 一键导出数据（JSON）
- 重置数据

## 🏗️ 项目结构

```
rungogo/
├── css/
│   └── style.css              # 卡通风格样式
├── js/
│   ├── storage.js             # 本地数据存储（localStorage）
│   ├── avatar.js              # 头像上传与处理

│   └── app.js                 # 主应用逻辑
├── index.html                 # 单文件版本（发布到 GitHub Pages）
├── index-standalone.html      # 构建产物（合并后的单文件）
└── build.py                   # 将 CSS/JS 合并为 index-standalone.html 的构建脚本
```

## 🚀 本地开发与构建

### 1. 克隆仓库
```bash
git clone https://github.com/patrickz2/rungogo.git
cd rungogo
```

### 2. 直接打开
由于项目为纯静态页面，可直接用浏览器打开 `index.html` 或 `index-standalone.html`。

### 3. 重新构建独立 HTML
```bash
python build.py
```
执行后会生成 `index-standalone.html`（包含内联 CSS 和 JS 的单文件版本）。

## ⚠️ 重要说明：数据存储方式

当前版本为**纯前端单机版**，所有数据（头像、昵称、打卡记录）保存在用户浏览器的 `localStorage` 中：

- ✅ 同一设备、同一浏览器再次打开，数据会自动保留
- ❌ 更换浏览器、清除缓存或换设备，数据会丢失
- ❌ 无法在不同用户之间共享数据，也无法由管理员查看他人记录

如需实现多人共享、云端同步、家长/管理员查看孩子打卡记录等功能，需要改造为后端版本（如微信小程序 + 云开发）。
